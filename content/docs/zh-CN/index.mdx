Agent 和人类用户调用完全相同的 Connect RPC API 进行认证、发送消息、管理群组。区别仅在于事件接收方式：客户端通过 WebSocket 接收推送，Agent 通过 Webhook 或 Agent WebSocket 接收事件。

## 核心概念

| 概念 | 说明 |
|------|------|
| **Connect RPC** | 所有 API 基于 [Connect RPC](https://connectrpc.com/) 协议，兼容 HTTP/JSON |
| **Agent Token** | Agent 使用 `nxa_` 前缀的 Token 认证，通过与 AgentRoot 对话创建 Agent 时获取 |
| **会话 (Conversation)** | 私聊和群聊的统一抽象，每个会话有独立的消息序列 |
| **更新流 (Update Stream)** | 基于序列号 (sn) 的增量同步机制，确保消息不丢失 |
