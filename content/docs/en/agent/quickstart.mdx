This guide walks you through creating an Agent that receives messages and replies automatically. By the end you will have a working bot inside Nexus AI.

## Prerequisites

- A Nexus AI user account (to interact with AgentRoot)
- Access to the Nexus AI API gateway (test environment: `https://api.nexus-dev.xsyphon.com`)
- A publicly accessible HTTPS endpoint (for Webhook), or ability to run a persistent process (for WebSocket)

## Step 1: Create an Agent via AgentRoot

The entry point for creating agents is AgentRoot — a built-in system Agent. Find AgentRoot in your client and start a conversation, then send:

```text
/newagent weather_bot Weather Bot
```

On success, AgentRoot returns:
- Agent basic info (username, display name, agent user ID)
- A one-time plaintext Token (`nxa_` prefix — save it immediately)

If you lose the Token, reset it (the old Token is invalidated immediately):

```text
/token weather_bot
```

### AgentRoot Commands

| Command | Description |
|---------|-------------|
| `/newagent <username> <display_name>` | Create Agent, returns one-time Token |
| `/myagents` | List your agents |
| `/agent <username>` | View Agent details (delivery mode, commands, etc.) |
| `/token <username>` | Reset Token (old Token invalidated immediately) |
| `/setwebhook <username> <url>` | Configure Webhook and return webhook secret |
| `/deletewebhook <username>` | Stop event delivery |
| `/websocket <username>` | Switch to WebSocket delivery mode |
| `/setcommands <username> <cmd:desc> ...` | Configure slash commands |
| `/help` | Show command help |

## Step 2: Verify Token

Call `UserService.GetProfile` to confirm your Agent identity:

```bash
curl -X POST https://api.nexus-dev.xsyphon.com/api.v1.UserService/GetProfile \
  -H "Content-Type: application/json" \
  -H "Connect-Protocol-Version: 1" \
  -H "Authorization: Bearer nxa_your_token_here" \
  -d '{}'
```

A successful response with your Agent's `userId`, `nickname`, and profile confirms the Token is valid.

## Step 3: Configure Event Delivery

Agents receive events in two ways. Pick one.

### Option A: Webhook (recommended for getting started)

Configure via AgentRoot command:

```text
/setwebhook weather_bot https://your-server.com/webhook
```

Or via API (called by the agent owner with their user access token):

```bash
curl -X POST https://api.nexus-dev.xsyphon.com/api.v1.AgentService/SetAgentConfig \
  -H "Content-Type: application/json" \
  -H "Connect-Protocol-Version: 1" \
  -H "Authorization: Bearer nxs_your_user_access_token" \
  -d '{"agentUserId": 1001, "deliveryMode": "AGENT_DELIVERY_MODE_WEBHOOK", "webhookUrl": "https://your-server.com/webhook"}'
```

On success, the response contains a `secret_key` for signature verification (HMAC-SHA256). Store it securely.

### Option B: WebSocket

Best for low-latency scenarios or when you cannot expose a public endpoint. See [WebSocket](/docs/agent/websocket).

## Step 4: Handle Webhook Events

Nexus AI sends JSON-serialized `Update` payloads:

```json
{
  "users": [
    {
      "userId": 42,
      "username": "alice",
      "nickname": "Alice",
      "avatarUrl": "https://...",
      "accountType": "ACCOUNT_TYPE_USER"
    }
  ],
  "groups": [],
  "snUpdate": {
    "sn": 1,
    "messageEnvelope": {
      "messageId": "1001",
      "conversationId": "8589934634",
      "senderId": 42,
      "body": {
        "type": "MESSAGE_TYPE_TEXT",
        "text": {"text": "Hello, Agent!"}
      },
      "createdAt": "1712000000000"
    }
  }
}
```

### Event Types

| Event Type | Description |
|------------|-------------|
| `MESSAGE` | `snUpdate.messageEnvelope` — New message (covers text, images, group events, and all other message types) |
| `CARD_ACTION` | `nonSnUpdate.cardAction` — User clicked an Adaptive Card Action.Submit button |
| `CONTACT_ADDED` | `snUpdate.contactAdded` — User added the Agent as a contact |
| `REMOVED_FROM_GROUP` | `snUpdate.removedFromGroup` — Agent was removed from a group |
| `GROUP_DISSOLVED` | `snUpdate.groupDissolved` — A group the Agent belongs to was dissolved |

## Step 5: Reply to Messages

Use the `SendMessage` API:

```bash
curl -X POST https://api.nexus-dev.xsyphon.com/api.v1.MessageService/SendMessage \
  -H "Content-Type: application/json" \
  -H "Connect-Protocol-Version: 1" \
  -H "Authorization: Bearer nxa_your_token_here" \
  -d '{
    "clientMessageId": "1",
    "conversationId": "8589934634",
    "body": {
      "type": "MESSAGE_TYPE_TEXT",
      "text": {"text": "Hello! I am your AI assistant."}
    }
  }'
```

Key fields:

| Field | Description |
|-------|-------------|
| `clientMessageId` | Client-generated unique ID for idempotent dedup |
| `conversationId` | From the received Webhook event |
| `body.type` | Message type — plain text here |
| `body.text.text` | Message content |

## Step 6: Complete Example (Python)

A minimal working Echo Agent:

```python
from flask import Flask, request, jsonify
import requests
import hmac
import hashlib

app = Flask(__name__)

AGENT_TOKEN = "nxa_your_token_here"
WEBHOOK_SECRET = "your_webhook_secret"
API_BASE = "https://api.nexus-dev.xsyphon.com"
msg_counter = 0

def verify_signature(body: bytes, timestamp: str, signature: str) -> bool:
    payload = f"{timestamp}.".encode() + body
    expected = hmac.new(
        WEBHOOK_SECRET.encode(), payload, hashlib.sha256
    ).hexdigest()
    got = signature.removeprefix("sha256=")
    return hmac.compare_digest(expected, got)

@app.route("/webhook", methods=["POST"])
def webhook():
    global msg_counter

    # Verify signature
    raw_body = request.get_data()
    ts = request.headers.get("X-Nexus-Timestamp", "")
    sig = request.headers.get("X-Nexus-Signature", "")
    if not verify_signature(raw_body, ts, sig):
        return jsonify({"error": "invalid signature"}), 401

    event = request.json

    # Handle new messages (snUpdate with messageEnvelope)
    sn_update = event.get("snUpdate")
    non_sn_update = event.get("nonSnUpdate")

    if sn_update and "messageEnvelope" in sn_update:
        msg = sn_update["messageEnvelope"]
        body = msg.get("body", {})
        if body.get("type") != "MESSAGE_TYPE_TEXT":
            return jsonify({"ok": True})

        user_text = body["text"]["text"]
        conversation_id = msg["conversationId"]

        msg_counter += 1
        requests.post(
            f"{API_BASE}/api.v1.MessageService/SendMessage",
            headers={
                "Content-Type": "application/json",
                "Connect-Protocol-Version": "1",
                "Authorization": f"Bearer {AGENT_TOKEN}",
            },
            json={
                "clientMessageId": str(msg_counter),
                "conversationId": conversation_id,
                "body": {
                    "type": "MESSAGE_TYPE_TEXT",
                    "text": {"text": f"You said: {user_text}"},
                },
            },
        )

    return jsonify({"ok": True})

if __name__ == "__main__":
    app.run(port=8080)
```

## Step 7: Verify Webhook Signatures

Every Webhook request includes signature headers:

| Header | Description |
|--------|-------------|
| `X-Nexus-Signature` | `sha256=<hex>` (HMAC-SHA256 signature) |
| `X-Nexus-Timestamp` | Request timestamp (Unix seconds) |

Verification:

```python
import hmac, hashlib

def verify_signature(body: bytes, timestamp: str, signature: str, secret: str) -> bool:
    payload = f"{timestamp}.".encode() + body
    expected = hmac.new(secret.encode(), payload, hashlib.sha256).hexdigest()
    got = signature.removeprefix("sha256=")
    return hmac.compare_digest(expected, got)
```

Also validate the timestamp — reject requests older than 5 minutes to prevent replay attacks.
