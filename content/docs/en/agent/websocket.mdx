Agent WebSocket provides low-latency real-time event delivery, ideal for scenarios requiring instant responses or when you cannot expose a public endpoint.

## Endpoint

```
wss://ws.nexus-dev.xsyphon.com/ws
```

## Authentication

The first frame after connection must be an auth request:

```json
{
  "requestId": "1",
  "type": "CLIENT_FRAME_TYPE_AUTH_REQUEST",
  "authRequest": {"token": "nxa_your_token_here"}
}
```

Server response:

```json
{
  "requestId": "1",
  "type": "SERVER_FRAME_TYPE_AUTH_RESPONSE",
  "authResponse": {"success": true, "userId": 1001}
}
```

On failure, `success` is `false` with an `error` field containing `errorCode` and `errorName`, and the connection is closed.

## Heartbeat

Send a heartbeat every 30 seconds after authentication:

```json
{
  "requestId": "2",
  "type": "CLIENT_FRAME_TYPE_HEARTBEAT_PING",
  "heartbeatPing": {}
}
```

The server disconnects after 90 seconds without a heartbeat.

## Receiving Events

Events arrive as `ServerFrame` messages with `type: SERVER_FRAME_TYPE_UPDATE` containing an `Update` payload. The `Update` uses the same format as Webhook вЂ” sequenced updates in `snUpdate`, non-sequenced in `nonSnUpdate`, with related entity info in `users` and `groups`:

```json
{
  "requestId": "0",
  "type": "SERVER_FRAME_TYPE_UPDATE",
  "update": {
    "users": [{"userId": 42, "nickname": "Alice"}],
    "groups": [],
    "snUpdate": {
      "sn": 1,
      "messageEnvelope": {
        "messageId": "1001",
        "conversationId": "8589934634",
        "senderId": 42,
        "body": {"type": "MESSAGE_TYPE_TEXT", "text": {"text": "Hello!"}},
        "createdAt": "1712000000000"
      }
    }
  }
}
```

Updates are sequenced вЂ” each `snUpdate` carries a monotonically increasing `sn`. Your agent can detect gaps by comparing the received `sn` with the last seen value and pull missed updates via `GetUpdates`.

Event types and fields are identical to Webhook. See [Webhook Events](/docs/agent/webhook) for details.

## Error Handling

The server may push error frames:

```json
{
  "requestId": "0",
  "type": "SERVER_FRAME_TYPE_ERROR",
  "error": {
    "error": {"errorCode": 1001, "errorName": "INVALID_TOKEN"},
    "fatal": true
  }
}
```

When `fatal` is `true`, the connection will close and the agent should reconnect.

## Reconnection Strategy

Use exponential backoff:

| Attempt | Wait |
|---------|------|
| 1 | 1s |
| 2 | 2s |
| 3 | 4s |
| 4 | 8s |
| 5+ | 30s (cap) |

Re-authenticate after each reconnection.

## Configuration

Switch to WebSocket mode via `SetDeliveryConfig`:

```bash
curl -X POST https://api.nexus-dev.xsyphon.com/api.v1.AgentService/SetDeliveryConfig \
  -H "Content-Type: application/json" \
  -H "Connect-Protocol-Version: 1" \
  -H "Authorization: Bearer nxa_your_token_here" \
  -d '{"websocket": {}}'
```

Also configurable via AgentRoot command:

```text
/websocket <agent_username>
```

This clears any existing Webhook configuration. The two modes are mutually exclusive.
