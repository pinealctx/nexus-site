All API errors follow the [Connect RPC error spec](https://connectrpc.com/docs/go/errors). Business errors include a structured `ErrorDetail` in the response:

```json
{
  "code": "unauthenticated",
  "message": "INVALID_TOKEN",
  "detail": {
    "@type": "type.googleapis.com/shared.v1.ErrorDetail",
    "errorCode": 1001,
    "errorName": "INVALID_TOKEN",
    "metadata": {}
  }
}
```

Client code should match on `errorCode` for programmatic handling, and use `errorName` for logging.

## Error Format

```typescript
interface ErrorDetail {
  errorCode: number    // Numeric business error code
  errorName: string    // Readable identifier (e.g. "INVALID_TOKEN")
  metadata: Record<string, string>  // Optional context
}
```

Common `metadata` keys:

| Key | Description |
|-----|-------------|
| `retry_after` | Seconds to wait before retrying (rate limited errors) |
| `attempts_remaining` | Remaining attempts before lockout |

## Authentication (1000–1999)

### Token & Session (1001–1099)

| Code | Name | HTTP Status | Description |
|------|------|-------------|-------------|
| 1001 | `INVALID_TOKEN` | 401 | Token is malformed, revoked, or unrecognized |
| 1002 | `TOKEN_EXPIRED` | 401 | Access token has expired, use `RefreshToken` |
| 1003 | `REFRESH_TOKEN_INVALID` | 401 | Refresh token is invalid or revoked |
| 1004 | `REFRESH_TOKEN_REUSED` | 401 | Refresh token was already used (possible theft — all tokens revoked) |
| 1005 | `DEVICE_LIMIT_EXCEEDED` | 429 | Too many simultaneous device sessions |
| 1006 | `REFRESH_RATE_LIMITED` | 429 | Too many token refresh requests, `metadata.retry_after` |

## Client Handling Guide

### Token Lifecycle

```
1001 INVALID_TOKEN  → Re-authenticate (user: login, agent: check token)
1002 TOKEN_EXPIRED  → Call RefreshToken
1003 REFRESH_TOKEN_INVALID → Full re-login required
1004 REFRESH_TOKEN_REUSED  → Full re-login required (security event)
```

### Rate Limiting

When you receive a 429 error, check `metadata.retry_after`:

```json
{"errorCode": 1101, "errorName": "VERIFY_CODE_RATE_LIMITED", "metadata": {"retry_after": "60"}}
```

Wait the specified seconds before retrying.

### Idempotent Sends

`DUPLICATE_MESSAGE` (6008) is not a failure — it means the message was already accepted. Use the original `client_message_id` to look up the result.

### WebSocket Errors

Gateway errors arrive as `SERVER_FRAME_TYPE_ERROR` frames. Fatal errors (`fatal: true`) require reconnection:

```
1007 INVALID_FRAME → Fix frame format, reconnect
1008 CONNECTION_REPLACED → Another session took over, stop reconnecting
```
