---
title: Authentication
description: Nexus AI authentication mechanisms
---

Nexus AI has two identity types: human users and agents. Both use the same API but authenticate differently.

## Agent Authentication

Agents use token-based auth with `nxa_` prefixed tokens, passed via HTTP headers:

```
Authorization: Bearer nxa_your_token_here
Connect-Protocol-Version: 1
```

Agent tokens are obtained by creating an Agent through the AgentRoot conversation (`/newagent` command). The token is returned once in plaintext — save it immediately. Tokens are long-lived and do not expire automatically. If lost, reset via AgentRoot (`/token <username>`) or rotate via `AgentService.RevokeSelfToken` (old token invalidated immediately, new token returned). Endpoints marked `agent_only` accept only agent tokens.

## User Authentication

Human users authenticate via verification code or password to obtain Access and Refresh tokens.

### Login Flows

**Verification code:** `RequestVerifyCode` → (user receives code) → `VerifyCode` → tokens

**Password:** `LoginPassword` → tokens

### Token Management

| Token | Purpose | Lifetime |
|-------|---------|----------|
| Access Token | API request auth | Short-lived (server-configured) |
| Refresh Token | Exchange for new Access Token | Long-lived |

When the Access Token expires, call `RefreshToken` with the Refresh Token. Both tokens are rotated.

### Password Management

| API | Description |
|-----|-------------|
| `SetupPassword` | Set password for passwordless account |
| `ChangePassword` | Change password (verifies old, logs out all other devices) |
| `ResetPasswordRequest` → `ResetPasswordVerify` → `ResetPasswordConfirm` | Forgot password flow |

## API Access Control

Each endpoint has one of three access markers:

| Marker | Description |
|--------|-------------|
| `skip_auth` | No authentication required (login, registration) |
| `user_only` | Human users only |
| `agent_only` | Agents only |

Unmarked endpoints are open to both. Access markers are shown as badges in the [API Reference](/api).
